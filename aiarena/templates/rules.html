General Rules

Bot authors must ensure that their creations are not harming the ladder system in any way, including but not limited to:

  • Slowing down the system on purpose
  • Generating huge amounts of data on the disc without purpose
  • Tempering with the filesystem or memory
  • Network access of any kind besides connecting to the local sc2api process is strictly forbidden
  • Uploading copies (clones) of another bot is forbidden

    • Bot Development

    The Ladder supports multiple programming languages
    To make sure the bot is accepted by the ladder you must follow the following standards:

    Python:
  • Version: Python 3.7
  • Bot called via run.py
    • Download example Bot.

    C++:
  • Can be compiled on Windows or Linux
  • Windows: Bot called via botname.exe
  • Linux: Bot called via botname
    • Download example Bot.

    Java:
  • Version: Java 10
  • Bot called via botname.jar
    • Download example Bot.

    DotNetCore:
  • Version: DotNetCore 2.1.403
  • Bot called via botname.dll
    • Download example Bot.

    GoLang:
  • Bot called via botname
    • Download example Bot.

    NodeJS:
  • Bot called via botname.js
    • Download example Bot.

    For a successful upload the Bot must be packed using zip. (Max 25 MB)
    The bots may write to a ./data directory, the data can be downloaded via profile page.

    Security

    The Ladder is completely automated and in theory attacks are quite simple to achieve. To counter this the games are played on seperated LadderServers which are firewalled and reset themselves after every game.
    Network access is denied and will lead to a permanent ban from the ladder.
    The training data is stored on another server and will be fetched before each game.

    Match Queue

    The match queue makes sure all Bots have to play against every other bot on the ladder. If the queue is empty a fresh one is generated containing all active bots. (Round-Robin) {% endblock %}
